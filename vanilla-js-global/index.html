vanilla-js-global

Sketch language JavaScript — type-checked via jsconfig checkJs; entry sketch.js via <script>
p5 mode global — globals provided by <script> tags
Type config vanilla-js-global/jsconfig.json — types: []. A local types.d.ts uses triple-slash reference directives: /// <reference types="p5/global" /> and /// <reference types="p5.transparency" />. These tell tsc to load the named packages' types via the normal node_modules resolution (node_modules/@types/<name> first, then node_modules/<name>'s package.json types field) — the same mechanism as adding them to tsconfig's types array.
p5 runtime CDN jsdelivr — cdn.jsdelivr.net/npm/p5@2.2.3/lib/p5.min.js, upstream (unpatched)
p5 types node_modules/p5 — with the pnpm patch applied; read only by tsc at type-check time, never executed
p5.transparency runtime CDN jsdelivr — cdn.jsdelivr.net/npm/p5.transparency@0.0.17/p5.transparency.min.js, upstream
p5.transparency types node_modules/p5.transparency — from local-clone tarball; rewritten .d.ts (read only by tsc)
Type patches p5, p5.transparency — p5 via pnpm patch; p5.transparency via rewritten .d.ts in the local-clone tarball
Runtime patches none — scripts load upstream CDN builds
Type-check ✓ pnpm run type-check:vanilla-js-global
Runs open index.html directly — or serve this folder; or pnpm run dev at project root and navigate to /vanilla-js-global/
Notes Type-check sees the patched shape; runtime uses upstream CDN. If upstream p5.transparency's const-reassignment path is hit at runtime it will throw — swap the CDN line for ../node_modules/p5.transparency/p5.transparency.js to exercise the local (fixed) build.
