vanilla-js-instance

Sketch language JavaScript — type-checked via jsconfig checkJs; entry sketch.js via <script>
p5 mode instance — new p5(sketch), methods called on the p parameter
Type config vanilla-js-instance/jsconfig.json — types: ["p5", "p5.transparency"]; a local types.d.ts provides the P5Instance alias used in JSDoc
p5 runtime CDN jsdelivr — cdn.jsdelivr.net/npm/p5@2.2.3/lib/p5.min.js, upstream
p5 types node_modules/p5 — with the pnpm patch applied; used only by tsc
p5.transparency runtime CDN jsdelivr — cdn.jsdelivr.net/npm/p5.transparency@0.0.17/p5.transparency.min.js, upstream
p5.transparency types node_modules/p5.transparency — from local-clone tarball; used only by tsc
Type patches p5, p5.transparency — p5 via pnpm patch; p5.transparency via rewritten .d.ts in the local-clone tarball
Runtime patches none — scripts load upstream CDN builds
Project workaround types.d.ts: P5Instance = InstanceType<Window["p5"]> — also var p5: Window["p5"], working around p5 v2's export as namespace p5 which leaves the UMD global as a module namespace with no construct signatures
Type-check ✓ pnpm run type-check:vanilla-js-instance
Runs open index.html directly — or serve this folder
Notes Requires node_modules to exist for type-check (not ideal for a truly standalone demo). See vanilla-js-instance-standalone for a fully self-contained instance-mode variant.
