vanilla-js-global-standalone

Sketch language JavaScript — type-checked via jsconfig checkJs; entry sketch.js via <script>
p5 mode global — globals provided by <script> tags
Type config vanilla-js-global-standalone/jsconfig.json — no types directive; include covers **/*.js and types/*.d.ts, so the bundled type files are picked up directly
p5 runtime CDN jsdelivr — cdn.jsdelivr.net/npm/p5@2.2.3/lib/p5.min.js, upstream
p5 types bundled in types/ — types/p5.d.ts + types/global.d.ts, copies of upstream 2.2.3, unpatched
p5.transparency runtime CDN jsdelivr — cdn.jsdelivr.net/npm/p5.transparency@0.0.17/p5.transparency.min.js, upstream
p5.transparency types bundled in types/ — types/p5.transparency.d.ts, global augmentations only
Type patches none — bundled type files are plain copies; no pnpm patch is consulted
Runtime patches none — scripts load upstream CDN builds
Type-check ✓ pnpm run type-check:vanilla-js-global-standalone
Runs open index.html directly — no tooling needed
Notes Works without the p5 type patch because global-mode augmentation happens via declare global { interface p5 }, which doesn't require p5's class to be a named export. Most portable variant; contrast with vanilla-js-instance-standalone, which needs a project-level type-level workaround to get the same end result in instance mode.
